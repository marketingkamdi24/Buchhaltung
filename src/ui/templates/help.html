Fetch API Data

Step 1: Fetch API Data
Configuration Options
  • Date Range: Enter dates in DD.MM.YYYY format (e.g., 21.10.2025)
  • Platforms: Toggle Amazon and/or eBay to fetch data from those platforms
  • Custom Origins: Add additional origin names if needed (comma-separated)
The API data includes ORDER_ID, KUNDENNR (KD-NR), and BELEGNR (RG-NR) for matching with your shop exports.
Step 2: Process Excel
File Requirements
  • Format: Excel files (.xlsx, .xls) or CSV
  • Structure: eBay shop export format with Bestellnummer in row 11
  • Size: Up to 50MB supported
What Gets Processed
  • Match Bestellnummer → ORDER_ID
  • Fill KD-NR from KUNDENNR
  • Fill RG-NR from BELEGNR
  • Calculate fee sums
  • Separate "Andere Gebühr" entries

Troubleshooting

Make sure to complete Step 1 (Fetch API Data) before uploading your Excel file. The API data is stored in your session and required for order matching.

  • Ensure the date range in Step 1 covers the dates of your shop export
  • Check that the correct platforms are selected
  • Verify your Excel file has the Bestellnummer column in the expected location (row 11)

  • Ensure file is .xlsx, .xls, or .csv format
  • Maximum file size is 50MB
  • Check that the file is not corrupted or password-protected
